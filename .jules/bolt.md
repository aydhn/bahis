## 2026-03-09 - [plot_animator.py: O(N^2) list comprehension overhead in PlotAnimator GIF loops]
**Learning:** Inside animation loops (`create_odds_gif`, `create_bankroll_gif`), repetitively extracting lists using comprehensions (`[d.get("val") for d in subset]`) causes major performance degradation as the data slice grows on each frame.
**Action:** Always pre-extract complete series data into `numpy` arrays *before* the loop, then use simple array slicing inside the loop (`all_values[:end_idx + 1]`). This yielded a ~35x speedup in iteration time. Furthermore, note that `matplotlib` figure saving with `bbox_inches="tight"` produces dynamically sized frames based on axis labels, which breaks `numpy.stack` in `imageio` since input arrays for GIF generation must have strictly uniform shapes. Remove `bbox_inches="tight"` to ensure consistent padding/dimensions across frames.
