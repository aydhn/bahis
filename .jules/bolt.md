## 2025-02-28 - Pytest Dependency Mocking
**Learning:** For unit testing quant finance modules in highly isolated CI/CD or restricted pipx environments, mocking core numerical dependencies (`sys.modules['numpy'] = MagicMock()`) creates brittle cascading failures if sub-modules or other core packages (like `pandas`) internally rely on exact versions or `import numpy`.
**Action:** Instead of dynamically mocking global dependency paths, either ensure the environment includes the dependencies (via `python3 -m pip install`) or refactor the tested module to remove unused/lazy numerical imports, ensuring a clean test run without forcing global `MagicMock` states that poison other modules.
