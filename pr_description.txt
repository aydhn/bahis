🧪 [testing improvement] Add comprehensive tests for LogRotator utility

🎯 **What:** The testing gap addressed
The `LogRotator` class located in `src/utils/log_rotator.py` was missing tests, creating a risk for log loss and unnecessary disk space usage if the logic broke. A full pytest suite has been added in `tests/utils/test_log_rotator.py`.

📊 **Coverage:** What scenarios are now tested
- Successful setup of active and archive log directories
- Archiving of uncompressed standard log files based on file age threshold (`archive_days`)
- Compression behavior (moving active `.log` to `archive/.log.gz`)
- Deletion of very old archived log files based on `max_archive_days`
- File name date pattern logic (`bot_YYYY-MM-DD.log`) override of standard file mtime threshold
- Edge case handling when `shutil.move` fails, validating error is captured properly in returned dictionary
- Log active stats reporting using `get_stats()` functionality

✨ **Result:** The improvement in test coverage
`LogRotator` is now fully covered by test cases mapping out its full operation lifecycle, verifying proper initialization, execution via active/archival flows, and data reporting without introducing any regressions in the codebase.
